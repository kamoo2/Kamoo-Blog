## 데이터베이스

> 사용자들이 공유하여 사용할 수 있도록 데이터를 통합하여 저장한 집합체

<hr/>

### 데이터베이스 정의
#### 1. 공유데이터
데이터베이스는 여러 사용자가 함께 사용하기 때문에 사용목적이 다른 사용자도 고려
어느 순간에도 둘 이상의 사용자가 동시에 데이터에 접근 가능

#### 2. 통합데이터
중복된 데이터를 최소화하는게 중요하고 통제 가능한 데이터만 중복을 허용 -> 데이터 불일치 현상을 없앨 수 있다.

#### 3. 저장데이터
데이터는 컴퓨터에서 주로 처리되므로 컴퓨터가 접근할 수 있어야 한다. -> 컴퓨터 저장장치에 저장된 데이터

#### 4. 운영데이터
서비스를 운영하기 위해 데이터는 지속적으로 유지되어야 한다. -> 임시로 저장된 데이터는 운영데이터가 아니다.

<hr/>

### 데이터베이스 특징
- `실시간 접근` : 사용자가 데이터 요청한 즉시 데이터를 전송
- `계속 변화` : 데이터베이스의 데이터는 어느 한 순간의 값을 나타내지만, 계속해서 바뀔 수 있다. CRUD 작업으로 바뀐 데이터를 저장하여 항상 최신의 데이터를 유지해야 함
- `동시공유` : 모든 사용자에게 동시에 제공되므로 같은 데이터에 동시에 접근할 수 있다.
- `내용에 따른 참조` : 데이터가 저장된 위치(주소)에 따라 참조하는 것이 아니라 데이터의 값에 따라 참조한다.

<hr/>

### 데이터 분류
- `정형 데이터` : 미리 정해진 구조(Schema)에 따라 저장된 데이터, 데이터 구조와 내용이 별도로 존재
- `비정형 데이터` : 소셜 서비스의 텍스트,음성,영상과 같이 정해진 구조가 없는 데이터로 최근 가장 증가하고 있는 데이터
- `반정형 데이터` : JSON,HTML과 같이 정해진 구조가 있지만 데이터 구조와 내용이 함께 존재

<hr/>

## DBMS

> Database Management System으로 데이터베이스 관리 시스템을 뜻한다.
> 사용자가 쉽게 데이터베이스에 접근,조작,제어할 수 있도록 도와주는 소프트웨어

<hr/>

### DBMS 주요기능
- `정의 기능` : 데이터베이스 구조를 정의하거나 수정할 수 있다.
- `조작 기능` : 데이터를 삽입,삭제,수정,검색하는 연산을 할 수 있다.
- `제어 기능` : 데이터를 항상 정확하고 안전하게 유지할 수 있다. 데이터를 여러 사용자가 공유해도 데이터 `일관성` `무결성`을 유지하며 장애가 발생해도 회복이 가능하게 하며 권한이 있는 사용자에게만 데이터 접근을 허용하여 보안을 유지하게 한다.

<hr/>

> DBMS에서 사용하는 언어에 대해 알아보자.

### DDL
데이터 정의어(Data Definition Language)로 데이터베이스 스키마를 정의할 때 사용하는 명령어
`CREATE` `DROP` `ALTER` `TRUNCATE`

### DML
데이터 조작어(Data Manipulation Language)로 데이터베이스에 저장된 데이터를 조회,수정,삽입,삭제하는 명령어
`SELECT` `INSERT` `UPDATE` `DELETE`

### DCL
데이터 제어어(Data Control Language)로 데이터베이스에 접근하고 사용하도록 권한을 부여하는 명령어
`GRANT` `REVOKE`

### TCL
트랜잭션 제어어(Transaction Control Language)는 말그대로 트랜잭션을 제어하는 명령어로 DCL의 일부로 분류하기도 한다.
`COMMIT` 트랜잭션 처리가 정상적으로 완료된 후 결과를 데이터베이스에 반영하는 명령어
`ROLLBACK` 트랜잭션 처리 과정에서 비정상적으로 종료된 경우 트랜잭션 내의 모든 연산을 취소하여 이전 DB 상태로 복구하는 명령어

<hr/>

## 삭제 관련 명령어 차이점
`DELETE` `TRUNCATE` `DROP`은 모두 데이터를 삭제하는 명령어지만 내부적으로 차이점을 가지고 있다.
- DELETE는 데이터를 지우지만 테이블 용량을 줄어들지 않고 원하는 데이터만 골라서 지울 수 있으며 삭제 후 되돌릴 수 있다.
- TRUNCATE는 테이블의 데이터를 모두 삭제(초기화)하는 명령어로 테이블 용량이 줄어들며 삭제 후 되돌릴 수 없다.
  데이터와 인덱스만 삭제되고 테이블을 삭제되지 않는다.
- DROP은 테이블 자체를 완전히 삭제하는 명령어로 삭제후 되돌릴 수 없다.
